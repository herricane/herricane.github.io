Algorithms: Elementary Sorts

Hetopia
2020-06-19
Algorithms
Sort

Selection Sort

Idea

  • In iteration i, find index min of smallest remaining entry.
  • Swap a[i] and a[min].

Invariants:

  • Entries to the left of the current pointer are fixed and in ascending order.
  • No entry to the right of the current pointer is smaller than any entry to the left of the pointer.

Implementation

Analysis

Selection sort uses (N - 1) + (N - 2) + … + 1 + 0 ~ N2 / 2 compares and N exchanges.

The running time is quadratic even if the input is sorted. The data movement is minimal: Linear number of exchanges are made in selection sort.

Insertion Sort

Idea

  • In iteration i, swap a[i] with each larger entry to its left.

Invariants:

  • Entries to the left of the pointer (including the pointer) are in ascending order.
  • Entries to the right of the pointer have not been seen.

Implementation

Analysis

To sort a randomly-ordered array with distinct keys, insertion sort uses ~ 1/4 N2 compares and ~ 1/4 N2 exchanges on average.

Best case: If the array is in ascending order, insertion sort makes N - 1 compares and 0 exchanges.

Worst case: If the array is in descending order (with no duplicates), insertion sort makes ~ 1/2 N2 compares and ~ 1/2 N2 exchanges.

For partially-sorted arrays, insertion sort runs in linear time. The number of exchanges equals the number of inversions (a pair of keys that are out of order).

Shellsort

Idea

  • We can move entries more than one position at a time by h-sorting the array.

An h-sorted array is h interleaved sorted subsequences. For example:

  • Shellsort is to h-sort array for decreasing sequence of values of h.

An example of shellsort:

  • To h-sort an array, we use insertion sort with stride length h.

This is because when the increments are big, the subarrays are small, so insertion sort will work well. And when the increments are small, the array is partially-sorted, so insertion sort will be fast.

  • Proposition: A g-sorted array remains g-sorted after h-sorting it.
  • Increment sequence:
    • 3x + 1: 1, 4, 13, 40, 121, 364, … (OK, easy to compute)
    • Sedgewick: 1, 5, 19, 41, 109, 209, 505, 929, 2161, 3905, … (Good)

Implementation

Analysis

The worst-case number of compares used by shellsort with the 3x + 1 increments is O(N3/2). Generally shellsort is useful in practice, since it is very fast and easy to implement.

Extra Question

Dutch national flag problem. Given an array of n buckets, each containing a red, white, or blue pebble, sort them by color. The allowed operations are:

  • swap(i, j): swap the pebble in bucket i with the pebble in bucket j.
  • color(i): determine the color of the pebble in bucket i.

The performance requirements are as follows:

  • At most n calls to color().
  • At most n calls to swap().
  • Constant extra space.

Approach. We use “0”, “1”, “2” instead of red, white and blue. Divide the array into four sections:

  1. a[0, ..., lo - 1] that stores 0s
  2. a[lo, ..., mid - 1] that stores 1s
  3. a[mid, ..., hi - 1] that stores unknown elements
  4. a[hi, ..., n] that stores 2s

At first the whole array is unknown. Using this approach, every time we encounter an unknown element, we can swap and store it to the corresponding section and increase/decrease related indices (lo, mid and hi). After only one traversal, the whole array should then be sorted (by color).
