Algorithms: Stacks and Queues

Hetopia
2020-06-16
Algorithms
Queue, Stack

Stack

Stack examines the item most recently added. (LIFO “last in first out”)

Linked List Implementation

Maintain pointer to first node in a linked list; insert/remove from front.

Array Implementation

Use array s[] to store N items on stack.

  • push[]: add new item at s[N].
  • pop[]: remove item from s[N-1].

Defect is that stack overflows when N exceeds capacity.

Considerations:

  • Null items: We allow null items to be inserted.
  • Loitering: Holding a reference to an object when it is no longer needed.

Resizing Array

  • Grow: Repeated doubling. If array is full, create a new array of twice the size, and copy items.
  • Shrink: Halve size of array when array is one-quarter full.

  • Performance: Starting from an empty stack, any sequence of M push and pop operations takes time proportional to M.

  • dequeue(): remove item from q[head].

Application

Goal: Evaluate infix expressions.

e.g. (1 + ((2 + 3) * (4 * 5)))

Dijkstra’s two stack algorithm: value stack, operator stack.

  • Value: push onto the value stack.
  • Operator: push onto the operator stack.
  • Left parenthesis: ignore.
  • Right parenthesis: pop operator and two values; push the result of applying that operator to those values onto the value stack.
