Mergesort

Idea

The basic idea of mergesort is:

  1. Divide the array into two halves.
  2. Recursively sort each half.
  3. Merge two halves.

To implement merging, we need to:

  1. Copy the array to an auxiliary array.
  2. Keep track of the indices in both halves.
  3. Compare the elements in those entries, put the smaller one back to the original array.
  4. Increment the index of the half that got one taken away. Repeat.

Analysis

  • Mergesort uses at most N lg N compares and 6N lg N array accesses to sort any array of size N. (Mathematical induction.)

  • Mergesort uses extra space proportional to N. (aux[] needs size N for the last merge.)

A sorting algorithm is in-place if it uses ≤ c log N extra memory (e.g. insertion sort, selection sort, shellsort).

Improvement

  • Use insertion sort for small subarrays. For example, cutoff to insertion sort for around 7 items.
  • Stop if the biggest item in the first half is smaller than the smallest in the second half.

Bottom-up Mergesort

  • Pass through the array, merging subarrays of size 1.
  • Repeat for subarrays of size 2, 4, 8, 16, …
  • No recursion needed.

Comparator

Comparator interface is used to sort using an alternate order. It must be a total order.

To support comparators in our sort implementations:

  • Use Object instead of Comparable.
  • Pass Comparator to sort() and less() and use it in less().

To implement a comparator:

  • Define a nested class that implements the Comparator interface.
  • Implement compare() method.

Stability

A stable sort preserves the relative order of items with equal keys. Insertion sort and mergesort are stable. Long-distance exchange might move an item past some equal item, resulting in being unstable.
